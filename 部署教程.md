# 考研背诵 — 部署教程（宝塔面板）

本文档面向部署人员，在 Linux 服务器的宝塔面板上完成项目部署。

---

## 环境要求

| 软件 | 版本 |
|---|---|
| Python | 3.10+ |
| Node.js | 18+ |
| MySQL | 5.7+（已在远端，无需安装） |
| 宝塔面板 | 最新版 |

---

## 1. 安装运行环境

### 1.1 安装 Python

宝塔面板 → **软件商店** → 搜索 **Python** → 安装 **Python 3.10+**。

安装完成后验证：

```bash
python3 --version
# Python 3.10.x

pip3 --version
# pip 24.x
```

### 1.2 安装 Node.js

宝塔面板 → **软件商店** → 搜索 **Node.js** → 安装 **Node.js 18+**。

安装完成后验证：

```bash
node --version
# v18.x 或更高

npm --version
# 9.x 或更高
```

### 1.3 配置 npm 镜像（可选，国内服务器推荐）

```bash
npm config set registry https://registry.npmmirror.com
```

---

## 2. 创建网站目录

```bash
mkdir -p /www/wwwroot/tongxiao
cd /www/wwwroot/tongxiao
```

把项目代码上传到这个目录。最终结构：

```
/www/wwwroot/tongxiao/
├── backend/
│   ├── run.py
│   ├── main.py
│   ├── config.py
│   ├── requirements.txt
│   ├── database/
│   └── migrations/
├── frontend/
│   ├── package.json
│   ├── vite.config.js
│   └── src/
└── .gitignore
```

---

## 3. 配置后端

### 3.1 创建虚拟环境

```bash
cd /www/wwwroot/tongxiao/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

**如果下载慢**，先配 pip 镜像：

```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

然后再 `pip install`。

### 3.2 检查数据库连接

确认 `backend/config.py` 中数据库配置正确：

```python
DB_TYPE = "mysql"
MYSQL_CONFIG = {
    "host": "119.45.182.166",
    "port": 9274,
    "user": "TongXiao",
    "password": "P8MtxmyYZMznkx8j",
    "database": "TongXiao",
}
```

---

## 4. 构建前端

```bash
cd /www/wwwroot/tongxiao/frontend
npm install
npm run build
```

构建完成后 `frontend/dist/` 目录出现即为成功。

---

## 5. 创建网站

浏览器打开宝塔面板，按以下步骤操作：

### 5.1 添加网站

1. 左侧菜单 → **网站** → 点击 **添加站点**

2. 填写：

   | 字段 | 值 |
   |---|---|
   | 域名 | 填写你的域名，如 `tongxiao.example.com` |
   | 根目录 | `/www/wwwroot/tongxiao/frontend/dist` |
   | FTP | 不创建 |
   | 数据库 | 不创建（使用远端 MySQL） |
   | PHP | 选 **纯静态** |

3. 点击 **提交**

### 5.2 配置反向代理

现在网站根目录是前端的静态文件，但 API 请求 `/api/*` 需要转发到后端。

1. 进入网站设置 → **反向代理** → 点击 **添加反向代理**

2. 填写：

   | 字段 | 值 |
   |---|---|
   | 代理名称 | `API` |
   | 目标 URL | `http://127.0.0.1:7896` |
   | 发送域名 | `$host` |
   | 高级功能 | 不填，保持默认 |

3. 点击 **提交**

这样 `/api/subjects` 会被转发到后端的 7896 端口，其他请求走 `dist/` 静态文件。

> 如果没有反向代理功能，也可以代码里处理，见 [附录 A](#附录-a直接启动不建网站)。

---

## 6. 启动后端服务

后端需要一直在后台运行。推荐宝塔的 **进程守护管理器**。

### 6.1 安装进程守护管理器

宝塔面板 → **软件商店** → 搜索 **进程守护管理器** → 安装。

### 6.2 添加守护进程

1. 打开进程守护管理器 → 点击 **添加守护进程**

2. 填写：

   | 字段 | 值 |
   |---|---|
   | 名称 | `TongXiao` |
   | 启动命令 | `/www/wwwroot/tongxiao/backend/.venv/bin/python /www/wwwroot/tongxiao/backend/run.py` |
   | 进程目录 | `/www/wwwroot/tongxiao/backend` |
   | 启动用户 | `root` |
   | 进程数量 | `1` |
   | 开机自启 | ✅ 勾选 |

3. 点击 **确定**

### 6.3 启动

点击守护进程列表中的 **启动** 按钮。看到以下日志说明成功：

```
[Flyway] ✓ V1__create_tables.py
[Flyway] ✓ V2__seed_data.py
INFO:     Uvicorn running on http://0.0.0.0:7896
```

---

## 7. 开放端口（如有防火墙）

### 7.1 宝塔安全组

宝塔面板 → **安全** → 添加端口 **7896** → 确定。

### 7.2 云服务商安全组（阿里云 / 腾讯云等）

登录云服务商控制台 → 安全组 → 入方向规则 → 放行 **7896** 端口。

---

## 8. 验证部署

浏览器打开 `http://你的域名`，应该看到：

- 📖 **背诵** — 科目列表
- 📥 **导入** — 文件导入页面
- 👤 **我的** — 统计数据

如果页面没有数据，检查：
1. 后端是否启动（进程守护管理器里状态是"运行中"）
2. 反向代理是否配置正确
3. `http://你的域名/api/subjects` 能否返回 JSON

---

## 9. 更新部署

后续发版时执行：

```bash
cd /www/wwwroot/tongxiao

# 拉取最新代码
git pull

# 更新后端依赖（如有新增）
cd backend
source .venv/bin/activate
pip install -r requirements.txt

# 重新构建前端
cd ../frontend
npm install
npm run build

# 重启后端
# 宝塔 → 进程守护管理器 → 重启 TongXiao
```

## 附录

### 附录 A：简化部署（直接启动，不建网站）

如果只是内部使用，不想配置宝塔网站：

```bash
cd /www/wwwroot/tongxiao/backend
source .venv/bin/activate
python run.py
```

然后浏览器打开 `http://服务器IP:7896` 即可。后端会自动托管前端静态文件。

### 附录 B：常见问题

| 问题 | 解决方法 |
|---|---|
| `ModuleNotFoundError: No module named 'xxx'` | 确认虚拟环境已激活：`source .venv/bin/activate`，重新 `pip install -r requirements.txt` |
| `Can't connect to MySQL` | 检查远端 MySQL 网络是否通，`config.py` 配置是否对 |
| 页面 502 Bad Gateway | 后端没有启动，检查进程守护管理器状态 |
| 页面打开空白 | `npm run build` 没有执行，或 `frontend/dist/` 不存在 |
| 8080 / 7896 端口无法访问 | 宝塔安全 + 云服务商安全组都要放行端口 |
