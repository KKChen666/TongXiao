## 1. 检查现有环境

在安装之前，先看看你的电脑上已经有哪些工具。有的话就跳过对应的安装步骤。

按 `Win + R`，输入 `cmd`，点确定，会打开一个黑色命令行窗口。

### 1.1 检查 Python

在命令行中输入：

```bash
python --version
```

**如果显示类似 `Python 3.12.3`：**
→ Python 已安装。再输入 `pip --version` 确认 pip 也可用。两个都有就 **跳过第 2 章**，直接看第 3 章。

**如果显示版本号小于 3.10（比如 `Python 3.8.x`）：**
→ 版本太旧，需要 **跟着第 2 章重新安装**。

**如果显示 `'python' 不是内部或外部命令`：**
→ 没有安装或没有配置 PATH，请 **跟着第 2 章安装**。

> 提示：你也可以试试 `python3 --version` 或 `py --version`，有些电脑上 Python 叫这个名字。

### 1.2 检查 Node.js

在命令行中输入：

```bash
node --version
```

**如果显示类似 `v20.11.0`：**
→ 已安装。再输入 `npm --version` 确认。都有就 **跳过第 3 章**。

**如果显示 `'node' 不是内部或外部命令`：**
→ 没有安装，请 **跟着第 3 章安装**。

## 2. 安装 Python

> ⚠️ 如果第 1 章检查时 Python 3.10+ 已经有了，直接跳到第 3 章。

### 2.1 下载

1. 打开浏览器，访问 [python.org/downloads](https://www.python.org/downloads/)
2. 页面会自动识别你的操作系统，点击黄色大按钮 **Download Python 3.x.x** 即可

### 2.2 安装

1. 🔒 **右键** 下载的 `python-3.x.x-amd64.exe`，选择 **「以管理员身份运行」**
2. ⚠️ **重要：** 勾选底部的 **「Add Python to PATH」**（这一步非常关键）
3. 点击 **「Install Now」**
4. 等待进度条走完，出现 "Setup was successful" 即为完成
5. 点击 **Close**

### 2.3 验证

关掉之前打开的命令行，重新按 `Win + R` → `cmd` 打开一个新的，输入：

```bash
python --version
# 输出示例: Python 3.12.3

pip --version
# 输出示例: pip 24.0
```

看到版本号说明安装成功。

## 3. 安装 Node.js

> ⚠️ 如果第 1 章检查时 Node.js 已经有了，直接跳到第 4 章。

### 3.1 下载

1. 打开浏览器，访问 [nodejs.org](https://nodejs.org/)
2. 点击左侧 **LTS** 版本下载（推荐长期支持版）
3. 下载 `node-vXX.X.X-x64.msi`

### 3.2 安装

1. 🔒 **右键** 下载的 `.msi` 文件，选择 **「以管理员身份运行」**
2. 点击 **Next**
3. 勾选 **「I accept the terms...」** → **Next**
4. 安装路径保持默认 → **Next**
5. 安装选项保持默认 → **Next**
6. **Install** → 等待完成 → **Finish**

> 安装程序会自动将 Node.js 添加到 PATH，无需手动操作。

### 3.3 验证

关掉命令行重开一个，输入：

```bash
node --version
# 输出示例: v20.11.0

npm --version
# 输出示例: 10.2.4
```

看到版本号说明安装成功。

## 4. 配置镜像源

国内网络访问 Python 和 Node.js 的官方源可能很慢（几十 KB/s），强烈建议花 1 分钟配好镜像。

### 4.1 pip 镜像（清华源）

打开命令行，执行：

```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

验证：

```bash
pip config list
# 应该看到: global.index-url='https://pypi.tuna.tsinghua.edu.cn/simple'
```

> 备选镜像：
> - 阿里云：`https://mirrors.aliyun.com/pypi/simple/`
> - 中科大：`https://pypi.mirrors.ustc.edu.cn/simple/`

### 4.2 npm 镜像（淘宝源）

执行：

```bash
npm config set registry https://registry.npmmirror.com
```

验证：

```bash
npm config get registry
# 应该看到: https://registry.npmmirror.com
```

> 备选镜像：
> - 腾讯云：`https://mirrors.cloud.tencent.com/npm/`

## 6. 配置 Python 虚拟环境

### 6.1 创建虚拟环境

1. PyCharm 窗口右下角，点击解释器名称（刚打开项目时显示 **`<No interpreter>`**）
2. 点击 **Add Interpreter** → **Add Local Interpreter**

3. 弹窗中：
   - 左侧选 **Virtualenv Environment**
   - 右侧选 **New**
   - Location：保持默认（`TongXiao\.venv`），不用改
   - Base interpreter：选你安装的 Python 3.10+

4. 点击 **OK**，等待底部进度条走完

### 6.2 安装 Python 依赖

在 PyCharm 底部点击 **Terminal** 标签。确认命令行开头有 `(.venv)` 字样，然后执行：

```bash
pip install -r backend/requirements.txt
```

正常输出：

```
Successfully installed fastapi-0.115.x pymysql-1.1.x uvicorn-0.34.x ...
```

| 包 | 用途 |
|---|---|
| `fastapi` | Web 框架，提供 REST API |
| `uvicorn` | ASGI 服务器，运行 FastAPI |
| `pymysql` | 连接远端 MySQL |

项目使用**远端 MySQL**，不需要本地装数据库。打开 `backend/config.py`，确认连接信息正确：

```python
DB_TYPE = "mysql"

    "host": "119.45.182.166",
    "port": 9274,
    "user": "TongXiao",

    "database": "TongXiao",
```

> 数据库已预先建好。首次启动时 Flyway 迁移会自动建表和填充数据。

## 9. 在 PyCharm 中运行

### 9.1 创建后端运行配置

1. 右上角点击 **Add Configuration...** → **+** → **Python**

2. 右侧填写：

   | 字段 | 值 |
   |---|---|
   | Name | `Backend` |
   | Script path | 点 📁 选 **`backend/run.py`** |
   | Python interpreter | 选 `.venv` |
   | Working directory | 选 `backend/` 目录 |

3. 点 **OK**

### 9.2 启动后端

右上角下拉选 **Backend** → 点 ▶ 启动。

弹出 Windows 防火墙提示的话，点 **允许访问**。

看到以下输出说明成功：

```
[Flyway] Applying V1__create_tables.py — create tables ...
[Flyway] ✓ V1__create_tables.py — XXms
[Flyway] Applying V2__seed_data.py — seed data ...
[Flyway] ✓ V2__seed_data.py — XXms
INFO:     Uvicorn running on http://0.0.0.0:7896
```

### 9.3 创建前端运行配置

1. 再打开 **Edit Configurations** → **+** → **npm**

2. 右侧填写：

   | 字段 | 值 |
   |---|---|
   | Name | `Frontend` |
   | package.json | 点 📁 选 **`frontend/package.json`** |
   | Command | `run` |
   | Scripts | `dev` |

3. 点 **OK**

### 9.4 启动前端

右上角选 **Frontend** → 点 ▶。看到以下输出说明成功：

```
  VITE vX.X.X  ready in XXX ms
  ➜  Local:   http://localhost:5465/
```

### 9.5 同时启动（推荐）

1. **Edit Configurations** → **+** → **Compound**
2. Name 填 `Full Stack`
3. 点下方 **+**，依次添加 `Backend` 和 `Frontend`
4. 点 **OK**

之后选 **Full Stack** → ▶，前后端一起启动。

## 12. 常见问题

### Q1: 启动后端报 `No module named 'fastapi'`

依赖没装。在 PyCharm Terminal 确认开头有 `(.venv)`，执行：
```bash
pip install -r backend/requirements.txt
```

### Q2: 启动后端报 `Can't connect to MySQL server`

连不上远端数据库。检查：
1. 网络能不能通（ping 那个 IP 试试）
2. `backend/config.py` 里的 host / port / user / password 对不对

### Q3: `npm install` 很慢

没配镜像。参考 [第 4 章](#42-npm-镜像淘宝源) 配一下。

### Q4: 页面开了但没数据

后端没启动。确认 http://localhost:7896/api/subjects 能打开看到 JSON 数据。

### Q5: 清空数据重来

在远端数据库执行：
```sql
DROP TABLE IF EXISTS flyway_schema_history, review_log, cards, topics, books, subjects;
```
重启后端，迁移会自动重建。

### Q6: Terminal 里没有 `(.venv)` 标识

点 Terminal 右上角 **+** 新建一个窗口，或手动执行 `.venv\Scripts\activate`。

### Q7: pip 安装报 SSL 错误

配了镜像后重试（[第 4 章](#41-pip-镜像清华源)），或执行：
```bash
pip install --trusted-host pypi.tuna.tsinghua.edu.cn -r backend/requirements.txt
```
